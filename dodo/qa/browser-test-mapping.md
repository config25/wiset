# W브릿지 AI 코칭 입력 자동화 — JSON → DOM 클릭 매핑 (브라우저 QA 지침)

> 브라우저 자동화(Claude 크롬 확장) 에이전트에게 그대로 넘기는 지침.
> 입력: 페르소나 JSON `{ user_profile, unstructured_data, consulting_log, ... }`.
> 목표: 5단계 위저드를 채우고 "AI 분석 시작하기"까지. JSP 구조에서 뽑은 정확한 selector 기준.

---

## 0. 전체 흐름 · URL · 공통 규칙

**흐름**: `/persona-select` → `/current-situation` → `/career-goal` → `/concern` → `/review` → `/analyzing`
- 모든 페이지에 **`?persona=N` 쿼리를 유지**한다 (N: `1`신규취업 `2`이직 `3`재취업 `4`승진). `user_profile` 맨 앞 "페르소나: {단계}"로 결정.
- 페이지 이동은 **URL 직접 이동보다 화면의 "다음 단계"(`#nextBtn`)** 를 눌러라. 선택값이 `sessionStorage`에 보관되므로 버튼 이동이 안전하다.

**공통 규칙**
1. 폼은 **개발 고정유저(userSn=1) 데이터로 이미 채워져 있다.** 각 섹션은 "기존 항목 삭제 → JSON 값으로 새로 입력"으로 교체.
2. JSON에 **없는 선택 섹션은 비운다(미입력).** placeholder(`○○○○`)나 남의 데이터는 삭제.
3. **`consulting_log`는 화면에서 입력하지 않는다.** DB 자동연동 값(리뷰에 "1:1 커리어컨설팅 N건 연동"으로 표시). 손대지 말 것.
4. 각 STEP 완료 후 캡처로 검증하고 다음으로.

---

## 1. STEP1 — 페르소나 선택 (`/persona-select`)

- 카드 선택: **`.persona-card[data-code="N"]`** 클릭 (또는 텍스트 `01 신규취업` / `02 이직 준비` / `03 재취업` / `04 승진 / 보직`).
- 다음: **`#nextBtn`**.

---

## 2. STEP2 — 현 상황 입력 (`/current-situation`)

`unstructured_data`를 `[섹션명]`으로 잘라 각 UI에 넣는다.

### 2-1. 학력/전공 (필수)
- 기존 항목 삭제: 각 학력 행의 **`[data-del]`** → 확인팝업 "삭제하시겠습니까?" 승인.
- 추가: **`#eduAdd`**("학력/전공 추가하기") → 모달 필드(`[data-f="..."]`):

| data-f | 라벨 | JSON 소스 (`[학력]` 줄: `석사 · {학교} · {학부}({전공}) ({졸업})`) |
|---|---|---|
| `se` (select) | 학력 구분 | 첫 토큰(학사/석사/박사/고등학교 졸업 …) |
| `school` | 학교명 | `{학교}` — **없으면 비움** |
| `entrance` | 입학년월 | (없으면 비움) |
| `graduation` | 졸업년월 | `({졸업})`의 연월 (예 `2026.08`) |
| `grad` (select) | 졸업상태 | 졸업 / 졸업예정 등 |
| `major` (필수) | 전공명 | `{학부}({전공})` (예: `정보보호대학원 (AI보안·머신러닝 전공)`) |
| `gpa` / `total` | 학점 / 총점 | `· 학점 4.21/4.50` → gpa=4.21, total=4.50 |
| `minor` | 다른 전공 | 복수/부전공 |
| `thesis` | 졸업논문/작품 | `논문: …` |

- 저장: **`[data-save]`**("저장").
- ⚠️ **학교명 칸에 전공을 넣지 말 것** (지난 실행의 실수). `석사 · 전자공학과 (제어·로보틱스 전공)`처럼 학교가 없으면 `school`은 비우고 `major`에 `전자공학과 (제어·로보틱스 전공)`.

### 2-2. 경력 (필수)
- 기존 삭제 → **`#carAdd`**("경력 추가하기") → 모달:

| data-f | 라벨 | JSON (`[경력]` 줄) |
|---|---|---|
| `company` | 회사명 | 회사명 |
| `dept` | 부서명 | 팀/부서 |
| `start` / `end` | 입사/퇴사년월 | 기간. **재직중이면 end 비움** |
| `position` | 직급/직책 | 선임연구원 등 |
| `jobField` | 직무 | (선택) |
| `salary` | 연봉 | (선택) |
| `desc` | 담당업무 서술 | `: ` 뒤 서술 |

- 저장: **`[data-save]`**.

### 2-3. 추가정보 섹션 (선택) — 버튼 `[data-modal="키"]`

**공통 패턴**: 버튼 클릭 → 모달 → (기존 항목 각각 `.wb-rec-x` 클릭 → "선택한 정보를 삭제하시겠습니까?" 승인) → `data-f` 채우고 **`[data-add]`("추가하기")** 를 항목마다 반복(그때그때 저장됨) → **`[data-x]`("완료")** 로 닫기.

| `[섹션]` | data-modal | 입력 필드(data-f) · 저장조건 |
|---|---|---|
| `[논문·연구]` | `paper` | `text`(제목+내용 한 줄). text 있으면 저장 |
| `[인턴·대외활동]` | `intern` | `kind`(select: 인턴/아르바이트/동아리/사회활동), `org`, `start`, `end`, `desc`. **kind≠선택 && org 있어야 저장** |
| `[교육이수]` | `edu2` | `name`, `org`, `start`, `end`, `desc`. **name 필수** |
| `[자격증]` | `cert` | `name`, `issuer`, `got`(취득일 0000.00.00), `exp`. **name 필수** |
| `[수상]` | `award` | `name`, `org`, `year`, `desc`. **name 필수** |
| `[해외경험]` | `overseas` | `country`, `start`, `end`, `desc`. **country 필수** |
| `[어학]` | `lang` | `lang`(select 외국어명), `speak`(select 회화능력), `testName`, `testScore`. lang≠선택 또는 testName 있으면 저장 |
| `[포트폴리오]` | `portfolio` | "URL 추가하기"로 URL 행 추가 / "파일 추가하기". URL 여러 개면 반복. **배치 저장 → "변경사항 저장"** |
| `[자기소개서]` | `cover` | 제목 + 내용(textarea) + 파일. |

- **어학 파싱**: `- 영어 · 비즈니스 회화 가능 · TOEIC 915` → `lang=영어`, `speak=비즈니스 회화 가능`(옵션에 맞게), `testName=TOEIC`, `testScore=915`.
- **없는 섹션**(예: 데이터에 [해외경험] 없음): 모달 열어 기존 항목만 삭제하고 완료 → 미입력.
- 다음: **`#nextBtn`**.

## 3. STEP3 — 경력개발 목표 (`/career-goal`) ⚠️ 실수 최다 구간

### 3-1. 희망 업종 `#industry` (select) — **카테고리 → 관련직 변환 필수**
`user_profile`의 산업은 **카테고리**인데 화면 드롭다운은 **관련직**이다. 아래로 변환해 `#industry` 값을 옵션 텍스트와 **정확히 일치**하게 설정:

| user_profile 카테고리 | `#industry`에 고를 관련직 |
|---|---|
| AI 정보보안 | **정보통신 관련직** |
| 화학 바이오 | **화학/식품가공 관련직** |
| 반도체 | **전기/전자 관련직** |
| 일반산업 | **기계 관련직** |

(전체 옵션: 생명 및 자연과학·교육·보건의료·디자인/방송·운전/운송·건축/토목 공학·기계·재료·화학/식품가공·전기/전자·정보통신·환경/에너지/안전·기술영업/판매·기타 관련직)

### 3-2. 희망 직무 `#job` (select)
`연구개발직` / `기술직` / `연구지원직` — `user_profile` "○○ 직무 희망" 또는 `target_role`의 뒷부분.

### 3-3. 희망 근무지 (복수)
- `#region1`(시도) 값 설정 → change 발생 → `#region2`(구군) 값 설정 → **`#regionAdd`("추가")** 클릭.
  - `서울 전체` = region1=`서울`, region2=`전체`. `인천 연수구` = region1=`인천`, region2=`연수구`.
- 기존 칩 삭제: 칩의 `[data-rm]`. 기본 '서울' 칩이 데이터에 없으면 삭제.

### 3-4. 희망 고용형태 (복수 토글) `#empRow` 안의 버튼
- 라벨과 일치하는 버튼을 클릭하면 ON(보라 테두리+연보라 배경+굵게). 여러 개면 각각 클릭.
- 옵션: `정규직 / 계약직 / 파견근로 / 대체인력 / 시간제 / 프리랜서 / 인턴직 / 무기계약직`.
- ⚠️ **"시간선택제"는 없음 → "시간제"로 매핑.** 클릭 후 **캡처로 ON 상태 확인**(지난 실행에서 여기서 헤맴).

### 3-5. 타겟 공고 `#targetList`
- DB 자동 스크랩(최대 3, 이미 선택됨). **그대로 둔다.**

- 다음: **`#nextBtn`**. ⚠️ 희망 업종·직무 **미선택이면 alert로 차단**되니 반드시 먼저 선택.

> 참고: STEP3는 페르소나 4(승진)면 화면이 다르다(`career-goal-growth`, 목표 보직/강화역량 등). 위는 1~3 공통. 4는 별도.

## 4. STEP4 — 세부 고민 (`/concern`)
- **`#freeText`**(textarea)에 `user_profile` "세부 고민:" 뒤 문장 전체 입력. (상단 탭/예시칩은 무시하고 직접 타이핑)
- 다음: **`#nextBtn`**.

---

## 5. STEP5 — 입력 데이터 확인 (`/review`)
- 01~04 섹션이 자동 렌더된다. 각 섹션 우측 **"수정"(`[data-edit="/경로"]`)** 으로 해당 단계 복귀 가능.
- ⚠️ **"미입력"으로 뜨는 필수/중요 항목**(특히 희망 고용형태·업종·직무·근무지, 페르소나)이 있으면 → 수정 버튼으로 돌아가 다시 채우고 재검토.
- 동의: **`#consentBox`** (기본 체크됨 — 그대로).
- 시작: **`#startBtn`("AI 분석 시작하기")** → `/api/analysis/save` POST → `/analyzing`. 이후 분석 진행바가 뜨면 완료까지 대기.

## 6. JSON → 화면 매핑 요약표

| JSON 위치 | 화면 |
|---|---|
| `user_profile` "페르소나:{X}" | STEP1 카드 + `?persona=N` |
| `user_profile` "신입/경력" | STEP2 경력단계 토글 |
| `user_profile` "{산업} 산업 희망" | STEP3 `#industry` (카테고리→관련직 변환) |
| `user_profile` "{직무} 직무 희망" | STEP3 `#job` |
| `user_profile` "희망 근무지: …" | STEP3 근무지 칩 |
| `user_profile` "희망 고용형태: …" | STEP3 `#empRow` 토글 (시간선택제→시간제) |
| `user_profile` "세부 고민:{…}" | STEP4 `#freeText` |
| `unstructured_data` `[학력]` | STEP2 학력/전공 (`#eduAdd`) |
| `unstructured_data` `[경력]` | STEP2 경력 (`#carAdd`) |
| `[논문·연구]` `[자격증]` `[어학]` `[인턴·대외활동]` `[교육이수]` `[수상]` `[해외경험]` | STEP2 추가정보 (`paper/cert/lang/intern/edu2/award/overseas`) |
| `[자기소개서]` `[포트폴리오]` | STEP2 `cover` / `portfolio` |
| `consulting_log` | 입력 안 함 (DB 자동연동) |

## 7. 남은 이슈(설계 결정 필요)

1. **업종 매핑 모호성**: 카테고리→관련직이 1:多(화학 바이오=2, 일반산업=10)라 §3-1 표는 "합의된 기본값"이다. 더 정확히 하려면 **테스트 JSON에 `screen_industry`(고를 관련직)를 같이 넣는 것**을 권장 — 그러면 에이전트가 추측 없이 `#industry` 옵션을 그대로 고른다.
2. **`취업우대/조건`**: `user_profile`엔 있는데 이 5단계 위저드엔 대응 입력 필드가 안 보인다(프로필 기존값일 가능성). **입력 대상 아님 → 무시**.
3. **`consulting_log`**: DB에 컨설팅 이력이 세팅돼 있어야 리뷰에 "N건 연동"으로 뜬다. 테스트 유저 DB 상태 확인 필요.
